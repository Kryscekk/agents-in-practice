# Contributing to Agents in Practice

Thanks for your interest! This repo is first and foremost **a public learning journal**, not a collaborative project in the usual sense. Here's what's welcomed, and how.

🇫🇷 [Lire en français](#contribuer-à-agents-in-practice)

---

## ✅ Very welcome

- **Issues** flagging an error, something broken, a confusing explanation
- **Pull requests** that fix a typo, a bug, or clarify an explanation
- **GitHub Discussions** to share how you adapted a tutorial to your case

## ⚠️ Needs discussion before PR

- Adding a new tutorial: open an issue first to discuss
- Major rewrite of an existing tutorial: same
- Translation (currently EN/FR, AR planned for month 5): open an issue to align

## ❌ Will be closed without debate

- PRs that make the tutorials "more pro" by losing the pedagogical aspect
- Adding frameworks (LangChain, LlamaIndex, etc.) — the repo is deliberately framework-free
- Massive re-architecture

## Style

- **Clear English/French** in the READMEs, no unexplained jargon
- **Simple Python code**, with helpful comments, no unnecessary abstractions
- **No exotic dependencies**: we stick to `anthropic`, `mcp`, `fastmcp`, `requests`, and the stdlib

## How to open an issue

Go to the **Issues** tab → **New issue**. Give the tutorial number concerned (e.g. `#09 MCP server`), describe what's happening, what you expected, and your OS/Python version.

Thanks 🙏

# Contribuer à Agents en Pratique

Merci de l'intérêt ! Ce repo est avant tout **un journal d'apprentissage public**, pas un projet collaboratif au sens habituel. Voilà ce qui est accueilli, et comment.

🇬🇧 [Read in English](#contributing-to-agents-in-practice)

## ✅ Ce qui est très bien accueilli

- **Issues** signalant une erreur, un truc qui ne marche plus, une explication confuse
- **Pull requests** qui corrigent une typo, un bug, ou clarifient une explication
- **Discussions** GitHub pour partager comment tu as adapté un tuto à ton cas

## ⚠️ Ce qui demande discussion avant PR

- Ajout d'un nouveau tuto : ouvre une issue d'abord pour qu'on en parle
- Réécriture importante d'un tuto existant : idem
- Traduction (EN/FR déjà disponibles, AR prévue au mois 5) : ouvre une issue pour s'aligner

## ❌ Ce qui sera fermé sans débat

- PR qui rendent les tutos "plus pros" en perdant l'aspect pédagogique
- Ajout de frameworks (LangChain, LlamaIndex, etc.) — le repo est volontairement sans framework
- Re-architecture massive

## Style

- **Français/anglais clair** dans les README, pas de jargon non expliqué
- **Code Python simple**, commenté, sans abstractions inutiles
- **Pas de dépendances exotiques** : on s'en tient à `anthropic`, `mcp`, `fastmcp`, `requests`, et la stdlib

## Comment proposer une issue

Va dans l'onglet **Issues** → **New issue**. Donne le numéro du tuto concerné (`#09 MCP server`), décris ce qui se passe, ce que tu attendais, et ton OS/version Python.

Merci 🙏
