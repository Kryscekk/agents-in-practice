# Contribuer à Agents en Pratique

Merci de l'intérêt ! Ce repo est avant tout **un journal d'apprentissage public**, pas un projet collaboratif au sens habituel. Voilà ce qui est accueilli, et comment.

## ✅ Ce qui est très bien accueilli

- **Issues** signalant une erreur, un truc qui ne marche plus, une explication confuse
- **Pull requests** qui corrigent une typo, un bug, ou clarifient une explication
- **Discussions** GitHub pour partager comment tu as adapté un tuto à ton cas

## ⚠️ Ce qui demande discussion avant PR

- Ajout d'un nouveau tuto : ouvre une issue d'abord pour qu'on en parle
- Réécriture importante d'un tuto existant : idem
- Traduction (EN/AR) : déjà planifiée à partir du mois 3/5, ouvre une issue pour s'aligner

## ❌ Ce qui sera fermé sans débat

- PR qui rendent les tutos "plus pros" en perdant l'aspect pédagogique
- Ajout de frameworks (LangChain, LlamaIndex, etc.) — le repo est volontairement sans framework
- Re-architecture massive

## Style

- **Français clair** dans les README, pas de jargon non expliqué
- **Code Python simple**, commenté en français, sans abstractions inutiles
- **Pas de dépendances exotiques** : on s'en tient à `anthropic`, `mcp`, `fastmcp`, `requests`, et la stdlib

## Comment proposer une issue

Va dans l'onglet **Issues** → **New issue**. Donne le numéro du tuto concerné (`#03 systemd`), décris ce qui se passe, ce que tu attendais, et ton OS/version Python.

Merci 🙏
